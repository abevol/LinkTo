# 规格说明书 (spec.md) - 数据迁移功能

## 1. 概述 (Overview)
在 LinkTo 中引入“数据迁移”模式。开启此功能后，用户定义的“源”文件或目录将首先被移动到“目标”位置，随后在原始位置创建一个指向新位置的链接。这主要用于释放原磁盘空间或重组文件结构，同时保持原路径的可访问性。

## 2. 需求分析 (Functional Requirements)

### 2.1 UI 变更
- 在“链接类型”分组框下方新增一个名为“数据迁移”的分组框。
- 该分组框标题处集成一个复选框（Checkbox），用于开启或关闭此功能。
- 分组框内包含一段简洁的功能描述：“开启后，将先移动源文件/文件夹至目标路径，再创建链接。”
- 该功能的状态应与现有的“创建链接”按钮逻辑联动。

### 2.2 核心逻辑
- **移动优先策略**：执行流程为：`检查目标 -> 移动文件/目录 -> 创建链接`。
- **原子性保障**：若移动成功但后续链接创建失败，系统必须尝试将文件/目录移回原始位置，确保不丢失数据且不留下破碎的路径。
- **冲突处理**：若目标路径已存在同名项，需弹出对话框提示用户选择“覆盖”、“跳过”或“取消”。
- **权限管理**：移动操作涉及系统路径或受限目录时，应能触发 UAC 提权或引导用户以管理员身份操作。
- **进度反馈**：针对大文件或多文件的目录移动，UI 应提供不阻塞的进度反馈（如进度条或状态文本）。

## 3. 非功能性需求 (Non-Functional Requirements)
- **可靠性**：严禁在移动过程中丢失用户数据。
- **响应性**：移动操作应在后台线程执行，避免 WinUI 界面失去响应。

## 4. 验收标准 (Acceptance Criteria)
- [ ] UI 上正确显示“数据迁移”分组框且布局美观。
- [ ] 勾选“数据迁移”后，执行创建操作，文件确实从 A 移动到了 B，且 A 处生成了正确的链接。
- [ ] 模拟链接创建失败（如权限不足），验证文件是否自动移回原处。
- [ ] 目标路径存在冲突时，确认对话框能正常工作。
- [ ] 大文件夹移动过程中，UI 不卡死且有进度显示。

## 5. 超出范围 (Out of Scope)
- 不处理跨网络驱动器的移动（暂仅支持本地磁盘）。
- 不涉及文件内容的修改或校验。
